1. 분석 방법

  • 데이터 수집: Playwright 기반 헤드드 브라우저로 사용자가 직접 로그인 후, 본인 권한 내 게시글만 수집.
  • 수집 필드: post_id, title, comment_count, author(익명화), created_at, view_count, like_count, post_url, body_text, raw_text, comments(옵션), page_number, collected_at, detail_status.
  • 페인포인트 판단 기준: 비용/고객/운영/인력/시설/플랫폼/창업·인수·폐업/경쟁/법·계약/심리적 부담 10대 카테고리. 사전 기반 매칭.
  • 익명화: 작성자명은 SHA1 해시 prefix(anon-xxxxxxxxxx)로 표시하며, HTML 리포트에는 원 작성자명을 노출하지 않음.
  • 한계: 사전 기반 매칭이므로 사전에 없는 표현은 누락될 수 있음. 원문 변형/요약은 수행하지 않으며 사전 외 의미는 추론하지 않음.

※ 본 리포트는 본인이 접근 권한을 가진 게시글만을 대상으로 한 UX 리서치 산출물이며, 원문은 변형 없이 그대로 표시됩니다. 작성자 식별 정보는 익명화되어 표시됩니다.
